## 第2章 Python解释器

Python程序运行过程：

1. 将原代码(（.py）编译为字节码（.pyc）；
2. 字节码在Python虚拟机（PVM）中运行。

如果原代码文件没有修改，程序第二次启动的时候会直接读取字节码文件，以提高运行效率。

将Python程序打包发布（也称为冻结二进制文件），就是将字节码、PVM和所需的支持文件打包在一个单独的文件中。常用的工具有：py2exe, PyInstaller, freeze。

## 第3章 如何运行程序

交互模式（Python Shell、IDLE）运行代码常用于学习、实验和测试代码。交互模式下输入复合语句，Shell的提示符从第二行开始由>>>变成...，IDLE从第二行开始自动缩进，使用一个空行结束复合语句。

包含Python语句的文本文件（.py）称为模块。运行一个模块的方式：

* 系统命令行方式：python script.py，或直接script.py

* 使用exec()运行模块代码：exec(open('mudule.py').read())
* 使用IDLE打开模块，选择Run Module运行模块。

使用内置函数dir()可以查看模块内部可用的变量名，其中以双下划线开头和结尾的变量是Python预定义的内置变量。

调试Python代码的方法：

* 阅读出错信息，修改相应代码
* 插入print语句
* 使用IDE GUI调试器
* 使用pdb命令行调试器
* 其他调试工具

## 第4章 Python对象类型

在Python中，数据以对象呈现，Python程序可分解为：

1. 程序由模块构成
2. 模块包含语句
3. 语句包含表达式
4. 表达式建立并处理对象

### 4.1 Python的核心数据类型

Python中处理的所有东西都是对象，其中内置对象（核心数据类型）和生成常量（literal）的语法如下：

* 数字——123，3.14，3+4j，Decimal，Fraction

* 字符串——'spam'，"guido's", b'a\xolc'
* 列表——[1,[2, 'three'], 4]

* 字典——{'food':'spam',   'taste':'yum'}
* 元组——(1, 'spam', 4,  'U')
* 文件——myfile=open('eggs', 'r')

* 集合——set('abc'), {'a', 'b',  'c'}
* 其他类型——类型、None、布尔型
* 编程单元类型——函数、模块、类

* 与实现相关的类型——编译的代码堆栈跟踪

其中，字符串、列表、元组为序列，即包含其他对象的有序集合。序列索引从0开始，用[]引用。也可以反向索引，负索引号从-1开始，关系为：正索引=len()+负索引。

其中，数字、字符串、元组是不可变的；列表、字典是可变的。

Python中没有类型声明，运行的表达式的语法决定了创建和使用的对象的类型。Python是动态类型的，也是强类型的。

Python核心数据类型的一个优秀的特性是它们支持任意的嵌套。

可用于多种类型的通用操作是Python内置函数或表达式；类型特定操作以方法调用的形式出现（如obj.method()）

列表解析表达式，是一种通过对序列中的每一项运行一个表达式来创建一个新列表的方法。其编写在方括号中，并且由使用了同一个变量名的表达式和循环结构组成。

字典是一种映射，存储键值对。

元组是不可变序列，提供了一种完整性约束。

一个Python核心概念：类型灵活性，不关注特定的类型。在Python中，编写对象的接口（支持的操作）而不是类型，这样，任何具有兼容接口的对象均能够工作，而不管它是什么对象类型。破坏代码灵活性的操作就是类型检查，至少有3种方法：

* if type(L)==type([]):
* if type(L)==list:
* if isinstance(L, list)

### 4.2 通用类型分类

相同分类的数据类型有很多相同的操作。

Python有三个主要的类型分类，对应三种类型的操作：

- 数字（整数、浮点数、二进制数、分数等），支持加法、乘法等
- 序列（字符串、列表、元组），支持索引、分片、合并等
- 映射（字典），支持通过键的索引等

集合是自成一体的一个分类。

Python核心类型又分为两类：

- 不可变类型（数字、字符串、元组、不可变集合），不支持原处修改，可以创建新的对象并分配给变量；
- 可变类型（列表、字典、可变集合），支持原处修改，不用创建新的对象。

不可变类型具有某种完整性，保证对象不会被程序的其他部分改变。

类型本身在Python中也是对象。内置函数type(X)返回对象X的类型对象，也可用isinstance()进行类型测试。

综合Python所有内置类型的分类：

| 对象类型  | 分类 | 是否可变 |
| --------- | ---- | -------- |
| 数字      | 数值 | 否       |
| 字符串    | 序列 | 否       |
| 列表      | 序列 | 是       |
| 字典      | 映射 | 是       |
| 元组      | 序列 | 否       |
| 文件      | 扩展 | N/A      |
| sets      | 集合 | 是       |
| frozenset | 集合 | 否       |
| bytearray | 序列 | 是       |

需要注意的几点：

* 对象根据分类共享操作
* 只有可变对象可以原处修改
* 数字报刊所有数字类型：整数、浮点数、复数、小数、分数
* 字符串包括：str、bytes；bytearray字符串类型是可变的
* 集合类似于一个无值的字典的键，既不是映射类型，也不是序列；frozenset是集合的一个不可变版本

Python中所有东西都是对象，类型（Type）本身也是对象。内置函数type(x)即返回x的类型对象。

## 第5章 数字

数字类型不是一个对象类型，而是一组类似类型的分类。

### 5.1 数字类型

数字类型包括：

* 整数和浮点数

* 复数，使用j或J表示虚部，可以使用内部函数complex(real, imag)创建

* 固定精度的十进制数

* 有理分数
* 集合
* 布尔类型
* 无穷的整数精度

* 各种数字内置函数和模块

整数可以自动支持无穷精度；浮点数使用双精度，可以使用科学计数标志e或E。

八进制、十六进制、二进制分别冠以：0o/0O, 0x/0X, 0b/0B；内置函数hex(I), oct(I), bin(I)把整数转换为3种进制表示的字符串，int(str, base)根据给定进制把字符串转换为整数。

### 5.2 表达式与操作符

表达式操作符及描述，并按优先级递升排列：

* yield x —— 生成器函数发送协议
* lambda args:   expression —— 生成匿名函数

* x if y else z —— 三元选择表达式

* x or y, x and y,   not x —— 或、与、非
* x in y, x not in y      ——成员关系（可迭代对象、集合）

* x is y, x is not  y —— 对象实体测试（内存地址意义上的相等）

* x<y, x<=y,  x>y, x>=y, x==y, x!=y —— 大小比较，集合子集和超集值相等性操作符
* x|y, x^y, x&y —— 位或、异或、与，集合并集、对称差、交集
* x<<y,      x>>y —— 左移或右移y位
* x+y, x-y —— 加法或合并，减法，集合差集
*  x*y, x%y, x/y, x//y —— 乘法或重复，余数或格式化，除法：真除法或floor除法

* -x, +x —— 一元减法，识别

* ~x —— 按位求补（取反）
* x**y —— 幂运算

* x[i] —— 索引（序列、映射及其他），点号取属性运算，函数调用
* x[i:j:k] —— 分片
* x(...) —— 调用（函数、方法、类、其他调用）

* x.attr —— 属性引用

* (...) —— 元组，表达式，生成器表达式
* [...] —— 列表，列表解析
* {...} —— 字典、集合、集合和字典解析

### 5.3 除法

有三种除法，两种运算符：

* x/y，传统除法和真除法，传统除法对于整数相除，省去小数部分，Python3已弃用；真除法均保留小数部分。
* x//y，Floor除法，也叫截断除法，省略小数部分，截断到真正结果之下的最近的整数。如果两个操作数中有一个浮点数，则返回浮点数，否则返回整数。与math.floor()结果相同，而math.trunc()向0截断，round()进行四舍五入。

### 5.4 数学工具

三种计算平方根的方法：

* 标准库函数：math.sqrt()
* 内置函数：pow(x, 0.5)
* 表达式：x**0.5

### 5.5 小数对象

小数是有固定精度的浮点数。通过导入模块decimal创建。用于固定精度应用（如货币）以及实现更好的数字精度。

这个模块的上下文对象可以指定精度和舍入方式等。设置全局精度：

    decimal.getcontext().prec=n

可以使用上下文管理器语句来设置临时精度：

    with decimal.localcontext() as ctx:
        ctx.prec=m

### 5.6 分数对象

分数对象实现了有理数对象，它保留了一个分子和一个分母，避免了浮点数的不精确性。

通过导入模块fractions，使用构造函数，传递一个分子和一个分母创建。

和小数对象一样，可以指定小数位数和舍入方式。

### 5.7 集合

集合（set）是一些唯一的、不可变的对象的无序集合（collection），集合的项是无序的、唯一的、不可改变的，支持与数学集合理论相对应的操作。

集合可以使用内置函数set()创建，传递一个序列或其他可迭代对象；也可以使用花括号来表述集合常量。空花括号创建一个空字典，所有空集合仍然需要使用内置函数创建。集合就像一个无值的字典的键集。

集合元素必须是可哈希的。所谓可哈希，就是一个元素有一个在其生命周期内不会改变的哈希值（内部有一个`__hash()__`方法），并且可以与其他元素进行比较（内部有一个`__eq()__`方法）。Python的不可变内建对象都是可哈希的，可变的容器（如列表和字典）都是不可哈希的。用户定义对象的实例都是可哈希的。

集合只能包含不可变（即可散列的）对象类型，因此，列表和字典不能嵌入到集合中，但元组可以。

集合本身是可变的，不能直接嵌入到其他集合中，但可以使用frozenset()创建一个不可变集合，嵌入其他集合中。

集合解析结构，编写在花括号中，运行一个循环并在每次迭代时收集一个表达式的结果，创建一个新的集合。例如：

    {x**2 for x in [1,2,3,4]}

集合的应用举例：

* 由于集合元素的唯一性，可以使用集合把重复项从其他数据集中过滤掉。collection ->      set -> collection
* 遍历图形或其他回环结构时，可以使用集合来记录已经访问过的位置
* 处理较大的数据集，如数据库查询

### 5.8 布尔类型

布尔数据类型为bool，其值为True和False，是预定义的内部变量名。

布尔数据类型实质上是int的子类，为其值重新定义了显示逻辑（str和repr的字符串格式）。

Python中所有对象都有真假概念：

* 数组如果非零则为真；
* 其他对象如果非空则为真。

Python的特殊对象`None`总为假。None代表某种对象，而非不确定，起到占位作用，类似C语言的NULL。

## 第6章 动态类型简介

Python语言中，类型是在运行时自动决定，而不是通过代码声明。

### 6.1 变量、对象和引用

Python语言中，变量的运行机制：

- 变量的创建。变量在第一次赋值时创建；
- 变量的类型。变量没有与之关联的类型信息和约束，它可以引用任何类型的对象。类型属于对象，而不是变量。变量只是在特定的时间引用了一个特定的对象。
- 变量的使用。表达 式中的变量会被其引用的对象代替，但必须在引用之前赋值。

变量和对象是独立存在的，通过连接相关联。从变量到对象的连接称为引用。变量总是连接到对象，但绝不会连接到另一个变量。

* 变量是一个系统表的元素，拥有指向对象的连接的空间；

* 对象是分配的一块内存，有足够的空间表示它所代表的值；

* 引用是自动形成的从变量到对象的指针。

每个对象有两个标准的头部信息：

* 一个类型标识符去标识对象的类型，即指向类型对象的指针；
* 一个引用计数器，记录当前指向该对象的引用数量，决定是否可以回收这个对象。称为垃圾收集。

### 6.2 共享引用

如果两个变量引用了同一个对象，叫共享引用。

Python中，变量总是一个指向对象的指针，而不是可改变的内存区的标签。给一个变量赋一个新值，不会改变原对象，而是让变量引用一个新的对象。

如果对象是可变的（在原处修改，如列表、字典以及自定义对象等），对一个变量的修改将会影响另一个变量。如果不希望这样，则需要拷贝对象而不是创建引用。如列表可以用切片的方法：

    L1=[1, 2, 3]
    L2=L1[:]

字典和集合可以使用对象方法X.copy()。通用的方法可以使用标准库：

    import copy
    Y=copy.copy(X)  #复制顶层对象
    Y=copy.deepcopy(X) #复制对象，包括嵌套对象

Python中有两种方法检查相等：

* X==Y，检查值相等
* X is Y，检查对象的同一性，即比较实现引用的指针，是检查共享引用的一种方法。如果两个变量引用同一个对象，则返回True。

如果是共享引用的两个变量X和Y，则X is Y == True。

Python为优化代码运行速度，对小的整数和字符串使用了缓存复用机制，即释放后不会被马上回收，而是等待被重复使用。使用以下代码查询一个对象被重复引用的次数：

    import sys
    sys.getrefcount(obj)

## 第7章 字符串

Python字符串为有序字符序列，为不可变类型，不能原处修改。字符串是一种序列类型（包括列表、元组等），支持序列类型的通用操作。

Python3中有3种字符串类型：str用于Unicode文本（ASCII或其他），bytes用于二进制数据（包括编码的文本），bytearray是bytes的一种可变的变体。

### 7.1 字符串常量

字符串常量的编写方法：

- 单引号：'spa"m'，单、双引号是等效的

- 双引号："spa'm"

- 三引号：'''...spam…'''，"""...spam…"""，用于多行的块字符串，也可废除多行代码
- 转义字符：\"s\tp\na\Om"
- Raw字符串：r"C:\new\test.spm"，也可用R，关闭转义机制，也用于正则表达式

- Python3中的Byte字符串：b'sp\x01am'

### 7.2 用转义序列代表特殊字节

转义序列使用反斜杠来引入特殊的字节编码。

Python提供的转义字符序列：

| 转义序列   | 意义             |
| ---------- | ------------------------ |
| `\newline`   | 忽视（连续）             |
| `\\ `        | 反斜杠                   |
| `\'`         | 单引号 |
|` \"`         | 双引号 |
| `\a`         | 响铃                     |
| `\b`         | 退格                     |
| `\f`         | 换页                     |
| `\n`         | 换行                     |
| `\r`         | 返回                     |
| `\t`         | 水平制表符               |
| `\v`         | 垂直制表符               |
| `\N{id}`     | Unicode数据库ID          |
| `\uhhhh`     | Unicode 16位的十六进制值 |
| `\Uhhhhhhhh` | Unicode 32位的十六进制值 |
| `\xhh`       | 十六进制值               |
| `\ooo`       | 八进制值（不大于3位八进制数，非编码类型字母o） |
| `\0`         | Null（不是字符串结尾）   |
| `\other`     | 不转义（其他情况下，保留反斜杠） |

注：

- \u和\U只能用于Unicode常量中，\u带4个十六进制数字，\U带8个十六进制数字，。
- 直接字符编码使用\xhh，2位十六进制数，或\ooo不大于3位八进制数。
- 如果\后不是一个合法的转义编码，则保留反斜杠

Python中，零（空）字符不会结束字符串，Python没有字符串结束字符。

### 7.3 字符串基本操作

字符串基本操作：

- `'abc'+'def'`，合并；
- `'abc'*8`，重复；
- `len(str)`，返回长度；
- `for x in Str: print(x)`，循环迭代；
- `c in str`，成员测试；

索引和分片操作：

- `S[i]`，索引S对象中的元素。Python允许使用负索引，索引位置等于字符串长度加负索引。
- `S[i:k]`，切片字符串，包含下边界，不包含上边界。上下边界缺省值为0和长度。`S[:]`得到S的一个拷贝。
- `S[i:j:k]`，扩展切片，从偏移i到偏移j-1，每隔k个元素索引一次。k取负值表示从右到左索引。S[::-1]反转字符串。

字符串转换：

- `int()`或`float()`，字符串转数字；
- `str()`，数字转字符串；
- `ord()`，返回字符的ASCII码或Unicode码；
- `chr()`，则相反

字符串是不可变序列，字符串的修改要使用合并、分片或str.replace()等字符串方法操作产生新的字符串对象。

如果原字符串空间被回收，新对象会重用原对象的空间。

为了修改一个超长字符串，可以将字符串转换成列表，修改后再转换成字符串。如：L=list(str), S=''.join(L)

### 7.4 字符串方法

字符串方法很多，举例常用方法：

`S.replace(old, new [, count])`，把S中的子串old替换为new。

`S.find(sub [, start [, end]])`，搜索子串的索引位置，如未找到，返回-1。

`S.join(iterable)`，用字符串连接一个可迭代对象的元素。

`S.split([sep [, maxsplit]])`，使用分隔符分割字符串。缺省的分隔符为空格、制表符和换行符。

文本解析就是分析结构并提取子串，可以使用分片或str.split()。

### 7.5 字符串格式化

Python的字符串格式化有两种形式：

- 字符串格式化表达式，基于C语言的printf模型，可能废弃。
- 字符串格式化方法，Python独有的方法，推荐使用。

#### 7.5.1 格式化表达式

格式化表达式使用操作符%：

1. 左侧为需要进行格式化的字符串，带有一个或多个嵌入的转换目标，以%开头；
2. 右侧为一个（或多个，嵌入到元组中）对象，这些对象会插入左侧一个或多个转换目标位置上。

例如：'There are %d %s bird!' % (2, 'red')

表达式左侧的格式转换目标的通用结构为：`%[(name)][flags][width][.precision]typecode`

字符串格式化表达式支持C语言中常规的printf格式化代码，但不像printf那样显示结果，而是返回结果。

| 代码    | 意义     |
| ---- | ------------------------ |
| s    | 字符串（或任何对象）     |
| r    | s，但使用repr，而不是str |
| c    | 字符                     |
| d    | 十进制整数               |
| i    | 整数                     |
| u    | 无符号整数               |
| o    | 八进制整数               |
| x    | 十六进制整数             |
| X    | x，但打印大写            |
| e    | 浮点指数                 |
| E    | e，但打印大写            |
| f    | 浮点十进制               |
| F    | 浮点十进制               |
| g    | 浮点e或f                 |
| G    | 浮点E或f                 |
| %    | 常量%                    |

类型转换代码在目标字符串的尾部，在%和转换代码之间，可以：

- 放置一个字典的键，即通过引用右边字典的键来提取值；
- 使用左对齐（-）、正负号（+）和补零（0）标志位；
- 给出数字的整体长度和小数点后的位数。with和precision可以使用*，指定从输入值的下一项中取值。

#### 7.5.2 格式化方法

使用主体字符串作为模板，接受任意多个参数，花括号通过位置或关键字指出替换目标及将要插入的参数，format方法传递待格式化对象。format方法也创建并返回一个新的字符串方法。如'{food},{0}'.format('spam',      food='eggs')。

字符串格式化方法在替换目标中的格式化结构为：`{fieldname!conversionflag:formatspec}`，其中

- fiieldname是指定参数的一个数字或关键字，后面为可选的.name或[index]成分引用，缺省为按顺序替换
- conversionflag可以是r, s, a，分别是该值上对repr, str, ascii内置函数的一次调用
- formatspec指定了如何表示该值，包括字段宽度、对齐、补零、小数点精度等，并以一个可选的数据类型编码结束。

formatspec的组成描述为：`[[fill]align][sign][#][0][width][grouping_option][.precision][typecode]`，其中

- fill为填充字符，如果指定对齐方式，可以任意字符填充，缺省为空格。
- align为<, >, =, ^，分别表示左对齐、右对齐、符号后填充、居中
- sign为+, -, space，分别表示正负都加符号、仅负数加符号、正数加空格负数加负号
- \#表示对整数、浮点、复数、固定精度浮点使用特定格式
- 0表示使用0填充位宽
- grouping_option，使用,或_千分位符号
- typecode为格式代码

其中格式代码有：

| 类型代码    | 意义                              |
| ---- | -------------------------------------------------------- |
| s    | 字符串，为字符串缺省格式                                 |
| b    | 二进制整数                                               |
| c    | 字符                                                     |
| d    | 十进制整数，为整数缺省格式                               |
| o    | 八进制整数                                               |
| x    | 十六进制整数，小写                                       |
| X    | 十六进制整数，大写                                       |
| n    | 除了本地格式外与d相同（）整数或与 g同（浮点）            |
| e    | 小写指数，缺省6位精度                                    |
| E    | 大写指数                                                 |
| f    | 浮点，缺省6位精度                                        |
| F    | 同f，nan to NAN, inf to INF                              |
| g    | 通用浮点格式，四舍五入后小数或指数显示，为浮点数缺省格式 |

内置函数format(value [, format_spec])，可以格式化一个单独的项，是上述方法的简单替代。

一些特定对象会定义自己的格式化代码，如datatime对象。





## 第8章 列表与字典

两种数据类型都可以在原处修改，可以按需增长或缩短，可以包含任何种类的对象或嵌套。

### 8.1 列表

列表是一种灵活的有序集合数据类型，列表的主要特性：

- 任意对象的有序集合，可以包含任何对象，列表元素是有序的；
- 通过偏移读取，支持分片、合并等操作；
- 可变长度、异构以及任意嵌套
- 属于可变序列的分类，支持原处修改、针对序列的操作
- 对象引用数组。列表的实现是指针数组，而不是链表；列表存储对象的引用，而不是对象的一个拷贝

### 8.2 列表应用

基本操作：

- `L1+L2`，连接
- `L1*n`，重复
- `for x in L: print(x)`，迭代
- `[f(x) for x in L]`，列表解析，对列表每一项应用一个表达式来构造一个新的列表
- `list(map(f(), L))`，与上式类似，对列表每一项应用一个函数，把结果收集在列表中
- `L[i]`，索引，对嵌套列表进行多次索引`L[i][j]`
- `L[i:j:k]`，分片

列表支持原处修改，需要将原处修改对象和生成新对象区分开。如果原处修改对象，会影响该对象的共享引用。

分片赋值的被赋值列表长度不需与分片长度匹配，所以分片赋值可以完成列表元素的替换、插入、删除等操作。

列表方法：

- `L.append(x)`，将x作为一个元素添加的列表L的末端，输出L+[x]
- `s.extend(t)` or s += t，用t的内容扩充s，相当于把两个列表连接起来，输出[…,t1,t2,...]
- `L.sort(key=func, reverse=False)`，对列表元素使用单参数函数，根据输出进行排序（升序或降序），可以使用内置函数sorted()
- `L.reverse()`，原地翻转列表，可以使用内置函数reversed()
- `L.pop()`，删除末端的元素
- `L.remove(v)`，通过值删除元素
- `L.insert(i, v)`，在偏移处插入值
- `L.index(v)`，查找值的偏移

注意：L.append()，L.sort()等方法对列表进行原处修改，返回一个None，所以L=L.append(x)将失去对列表的引用。

可以是del语句删除列表元素或分片，如del L[i]或del L[i:j]

空列表赋值给分片会删除分片，空列表赋值给一个索引值，会在该位置插入空列表。

### 8.3 字典

字典，又叫关联数组或散列表，其主要特性：

- 通过键而不是偏移量来读取
- 任意对象的无序集合
- 可变长度、异构、任意嵌套
- 属于可变映射类型，唯一的内置映射类型，通过键映射到值的对象
- 对象引用表，支持键读取的无序对象引用表，通过散列表实现。

### 8.4 字典应用

基本操作：

- D[key]，用键索引值
- len(D)，字典长度，与len(D.keys())相同
- key in D，键测试，字典的in成员关系操作符，提供了键存在与否的测试方法，
- D[key]=value，字典键值修改，对字典中没有的键进行赋值会创建一个新的元素
- del D[key]，删除键值
- for k in D: print(D[k])，迭代，字典有自己的迭代器，返回连续的键，与for k in D.keys()相同

注意keys(), values(), items()返回一个可迭代的视图对象，而不是列表，需要调用list()生成列表。

字典方法：

- D.keys()，返回所有的键列表

- D.values()，返回值列表

- D.items()，返回键值元组列表

- D.get(key)，返回值，类似键索引，读取不存在的键时会报错（missing-key），而get()方法能够返回默认值（None或用户定义的默认值）

- D.update(D2)，把D2键值合并到D1，会覆盖D1中相同的键
- D.pop(key)，删除一个键

使用字典的注意事项：

- 序列运算无效
- 对新索引赋值会添加项
- 任何不可变对象都可以作为键

字典应用实例：

- 使用数字作为键，模拟灵活的列表
- 使用元组作为键，用于稀疏数据结构，如：Matrix={},Matrix[(2,3,4)]=99

避免missing-key错误的方法：

- 预先对键进行测试，if (2,3,6) in Matrix:
- 使用try语句捕获并修复这一异常，try: Matrix[(2,3,6)] except KeyError:
- 使用get()方法，Matrix.get((2,3,6),      0)

使用字典存储结构化信息，实现其他语言的记录或结构。

### 8.5 创建字典的方法

创建字典的方法：

- {key:value,...}，字典常量表达式
- D[key]=value，新键赋值，动态建立
- dict(key=value,...)，字典构造函数，键必须是字符串，不需要引号
- dict([(key,  value), ...])，键值元组组成的序列
- dict.fromkeys(keys[, value])，使用键列表，使用一个值初始化，缺省为None
- dict(zip([key,...], [value,...]))，zip函数
- {k:v for (k, v) in  [(k1,v1),...]}，字典解析表达式
- {k:v for (k, v) in  zip(keys, values)}，字典解析表达式和zip函数结合

### 9.1 元组

元组的主要特性：

### 9.2 文件

打开文件，使用内置函数open(file, mode='r')，处理模式包括：r输入（默认）、w输出、a追加，模式字符后加b进行二进制处理、加+进行读写。

文件的基础用法：

- 文件迭代器是最好的读取行工具：for line in  open('data'): use line
- 内容是字符串，而不是对象，读入时按照需要转换成其他对象，输出时需要格式化字符串。
- 虽然有自动回收机制，但手动关闭文件是一个好的做法
- 默认情况下，输出文件总是缓存的，关闭文件或调用flush()方法，迫使缓存数据写入硬盘
- 文件是可查找的。Python文件是在字节偏移基础上随机访问的，调用seek方法跳转到指定位置读写。

### 9.3 文件应用

文本文件与二进制文件的区别：

- 文本文件把内容表示为常规的str字符串，自动执行Unicode编解码，并默认执行末行转换。所有文本都是Unicode
- 二进制文件把内容表示为一个特殊的bytes字符串类型，并且允许程序不修改地访问文件内容

使用内置函数eval(expr)，将文件读入的表达式字符串解析为Python对象。

pickle模块可以实现对象序列化，直接在文件中存储几乎任何Python对象。

```python
import pickle

file=open('data.pkl', 'wb')  # 以二进制打开文件
pickle.dump(obj, file)  # 序列化对象
pickle.load(file)  # 解析对象
```

文件的上下文管理器允许把文件处理代码包装到一个逻辑层中，以确保退出后自动关闭文件：

```python
with open() as file:
	for line in file:
		use line
```

try/finally语句提供类似的功能，但要额外的代码：

```python
file=open()
try:
	for line in file:
		use line
finally:
	file.close()
```

Python中其他文件工具：

- 标准流，在sys模块中预先打开的文件对象，如sys.stdout
- os模块中的描述文件，处理整数文件
- sockets, pipes, FIFO文件，文件类对象，用于同步进程或通过网络进行通信
- 通过键直接存取的文件
- shell命令流，像os.popen和subprocess.popen这样的工具，支持产生shell命令，并读写到标准流
- 第三方扩展文件类，如pyserial, pexpect等

## 第14章 迭代器和解析，第一部分

### 14.1 迭代器初探

可迭代对象，也称迭代器对象，或迭代器，是序列概念的通用化，如果对象是实际保存的序列，或可以在迭代工具环境中（如，for循环）一次产生一个结果的对象。包括实际序列和按照需求而计算的虚拟序列。可迭代对象自动创建了`__next__`方法。

迭代工具包括for循环、列表解析、in成员关系测试、map、sorted、zip内置函数等。迭代工具的工作机制是每次迭代中调用`__next__`方法，并捕捉StopIteration异常以确定何时离开。

Python迭代协议：每次调用`__next__`方法会前进到下一个结果，在末尾时引发StopIteration异常。这是迭代工具的内部工作机制。实现了迭代协议的任何工具，都能够在可迭代的内置类型或用户自定义类上自动地工作。

手动迭代：

- 内置函数iter(*object*[, *sentinel*])返回对象的迭代器，其中包含`__next__`方法。
- 内置函数next(*iterator*[, *default*])调用可迭代对象的`__next__`方法，等价于`X.__next__()`。

### 14.2 列表解析初探

列表解析对迭代对象的每个元素执行一个操作，生成一个新的列表。其一般表达式：

[表达式 for 循环变量 in 迭代对象 if 条件表达式]

其中可以包含任意数目的for/if子句。

列表解析把一个表达式映射遍一个序列，map()函数把一个函数映射遍一个序列。

列表解析可以处理矩阵（嵌套列表），如提取列，`[row[j] for row in M]`；提取对角线元素，`[M[i][i] for i in range(len(M))]`

就性能而言，map()调用比等效的for循环快两倍，而列表解析往往比map()调用稍快一些。

一些迭代工具的例子：

- `map(function, iterable, ...) `，对可迭代对象的每个元素应用函数，返回一个可迭代对象。
- `sorted(iterable[, key][, reverse])`，返回可迭代对象每个元素的排序列表
- `enumerate(iterable, start=0)`，根据相对位置配对可迭代对象中的项，返回一个可迭代对象。
- `filter(function, iterable) `，应用函数对可迭代对象的每个元素进行过滤，返回一个可迭代对象。
- `zip(iterables)`，组合可迭代对象的各项，返回可迭代对象。
- 函数的*arg参数，接受可迭代对象，并解包为单个参数。如zip(*zip(X,Y))把zip的元组序列unzip。

部分可迭代对象有自己的迭代器，它们只支持单个迭代器，如map、zip、filter返回对象；部分可迭代对象没有自己的迭代器，可以使用iter手动生成多个迭代器，每个迭代器都会记住自己的位置，如range，I1=iter(range(n)); I2=iter(range(n))。

## 第17章 作用域

Python使用变量都是在一个命名空间（一个保存变量名的地方）中进行的，变量赋值的位置决定了变量的作用域。

根据赋值位置，变量的三个作用域：

* 在def内赋值，作用域为函数内
* 在嵌套的def内赋值，对于嵌套的函数，变量是非本地的
* 在def外赋值，作用域是文件全局的

函数定义了本地作用域，模块定义了全局作用域。两个作用域（命名空间）的关系法则：

- 每个模块都是一个全局作用域。外部的全局变量成为一个模块对象的属性。
- 全局作用域的作用范围仅限于单个文件。全局作用域对应模块。
- 每次对函数的调用都创建了一个新的本地作用域。本地作用域对应函数调用。
- 函数内部赋值的变量名除非声明为全局变量或非本地变量，否则均为本地变量。
- 所有其他的变量名都可以归纳为本地、全局或者内置的。
- 交互模式工作在内置模块`__main__`中，遵循同样的作用域规则。
- 函数中任何类型的赋值都会把一个名称划分为本地的，包括=赋值，import模块名，def函数名和参数
- 原处修改对象并不会把变量划分为本地变量，只有变量名赋值才可以。对象和名称的区别：修改一个对象并不是对一个名称赋值。

Python变量名解析的LEGB原则：

- 当在函数中使用未认证的变量名时，Python搜索4个作用域：本地作用域（L）、上一层结构中def或lambda的本地作用域（E）、全局作用域（G）、内置作用域（B），并在第一次找到就停止搜索。
- 当在函数中给一个变量名赋值时（而不是在表达式中引用），Python会创建或改变本地作用域的变量名，除非它已经声明为全局变量。
- 当在函数之外给一个变量名赋值时（在模块文件的顶层或在交互模式下），本地作用域和全局作用域（这个模块的命名空间）是相同的。

对于一个def语句，全局声明或非本地声明改变默认的变量搜索顺序，将赋值的变量名映射到模块文件内部的作用域或嵌套函数作用域。

内置作用域仅仅是一个名为`__builtins__`的内置模块，定义了Python的内置异常、常量和函数，Python运行时会自动导入这个模块，也可以手动导入builtins模块。

global语句是一个命名空间的声明，在函数内声明一个或多个全局变量：

- 全局变量是位于模块文件内部的顶层的变量
- 全局变量如果在函数内部被赋值的话，必须经过声明
- 全局变量在函数内部不经过声明也可以被引用。

全局变量是Python中保持状态信息的最直接的一个方法，好的做法是用单个的模块文件定义所有的全局变量。

一个模块文件的全局变量一旦被导入就成为这个模块对象的一个属性。但文件间进行通信最好的办法就是通过调用函数（传递参数、得到返回值）。

使用全局变量的三种方法（假设在thismodule.py中定义了一个全局变量var）：

* global var
* thismodule.var
* sys.modules['thismodule'].var


嵌套作用域，也叫静态嵌套作用域，如果一个变量在函数内部声明为nonlocal，则赋值会修改最近的嵌套函数的本地作用域的变量，使对变量的查找略过本地作用域，从嵌套作用域开始，并且限制在嵌套作用域内。

工厂函数，也叫闭合（closure），一个能够记住嵌套作用域的变量值的函数，尽管那个作用域或许已经不存在了。例如：

```python
def maker(N):
    def action(X):
        return X**N
    return action
```
类更适合用作记忆状态。如果不使用类，全局变量，嵌套作用域引用，默认参数，函数属性是函数保留状态信息的主要方法。

对于带有循环变量的嵌套作用域，如果def或lambda在函数中定义，并嵌套在一个循环中，且引用了循环变量，则循环中产生的函数将会引用最后一次循环完成时的变量值。这是因为嵌套作用域中的变量在嵌套的函数被调用时才进行查找。为了使代码工作，必须使用默认参数把当前的值传递给嵌套作用域的变量。如下例所示：

```python
def makeActions():
    acts=[]
    for i in range(5):
        #acts.append(lambda x: i**x)  # remember the last i
        acts.append(lambda x, i=i: i**x)
    return acts
```

使用nonlocal声明的变量名可以在嵌套函数内进行修改，同时也加快了变量的引用（它忽略的本地作用域）。例如：

```python
def tester(start):
    state=start
    def nested(label):
        nonlocal state
        print(label, state)
        state+=1
    return nested  # 返回的嵌套函数记住了外层函数的状态state
```

在一个函数内使用时，global和nonlocal限制了查找规则：

- global使得作用域查找从嵌套模块开始，直到内置作用域。但赋值总是在模块作用域内。

- nonlocal使得作用域查找限制在嵌套作用域内，要求名称必须已经存在，也不会继续到全局或内置作用域内。赋值总是在嵌套作用域内。


需注意：和global不同，执行nonlocal语句时，变量必须已经在嵌套的作用域中赋值过。

## 第18章 参数

Python函数传递概念：

* 参数的传递是通过自动将对象赋值给本地变量名来实现的。
* 在函数内部的参数名的赋值不会影响调用者。
* 改变函数的可变对象参数的值也许会对调用者有影响。

函数调用过程中的参数传递，是将对象赋值给函数本地变量的过程，是通过对象引用而不是拷贝传递。。因为引用是以指针形式实现的，所以参数实际上是通过指针进行传递的。实际效果为：

- 不可变对象参数“通过值”进行传递。
- 可变对象参数“通过指针”进行传递。如果不想在函数内部修改调用者的对象，可以明确创建一个可变对象的拷贝。

函数参数的匹配模型：

- 位置：从左到右进行匹配
- 关键字参数：通过参数名进行匹配，name=value
- 默认参数：为没有传入值的参数定义参数值，name=value
- 可变参数：收集任意多基于位置或关键字的参数，在函数定义头部以*开头，收集任意多的参数
- 可变参数解包：传递任意多基于位置或关键字的参数，调用者使用*语法将参数集合打散，传递任意多的参数
- Keyword-only参数：参数必须用带有关键字参数的名称来传递。出现在`*args`之后，`**args`之前，也可以使用一个单独的*，表示不接受任何可变参数，只表示其后为Keyword-only参数。

注：*形式创建或解包元组；**形式使用关键字参数，创建或解包字典。

函数参数匹配顺序法则：

- 函数调用中：位置参数（value），关键字参数（name=value）和`*sequence`形式的组合，`**dict`形式
- 函数定义头部：一般参数（name），默认参数（name=value），`*name`形式（或`*`），name或name=value形式keyword-only参数，`**name`形式。

注：在函数调用和函数定义头部，如果出现`**args`形式的话，都必须出现在最后，不能单独使用`**args`参数。

Python内部使用以下步骤进行参数匹配：

1. 通过位置分配非关键字参数
2. 通过匹配变量名分配关键字参数
3. 其他额外的非关键字参数分配到`*name`元组中
4. 其他额外的关键字参数分配到`**name`字典中
5. 用默认值分配给在头部未得到分配的参数

`*`和`**`让函数支持接受任意数目的参数，都可以出现在函数的定义和调用中。

在函数定义中，将所有不匹配的位置参数收集到一个元组中：

```python
def f(*args):
    print(args)
f(1,2,3)  # args=(1,2,3)
```

与`*`类似，`**`只对关键字参数有效，将所有不匹配的关键字参数收集到一个字典中：

```python
def f(**args):
    print(args)
f(a=1, b=2, c=3)  # args={a=1, b=2, c=3}，关键字会传入函数
```

混合使用一般参数、`*`参数和`**`参数：

```python
def f(a, *pargs, ***kargs):
    print(a, pargs, kargs)
f(1, 2, 3, x=4, y=5)  # a=1, pargs=(2, 3), kargs={x=4, y=5}
```

在函数调用中，`*`会解包参数的集合：

```python
def f(a,b,c):
    print(a,b,c)
args=[1,2,3]
f(*args)  #  a=1, b=2, c=3
```

`**`会以键值对解包一个字典，形成关键字参数：

```python
def f(a,b,c):
    print(a,b,c)
args={'a':1, 'b':2, 'c':3}  # 关键字必须与函数定义一致
f(**args)  #  a=1, b=2, c=3
```

keyword-only参数必须按照关键字传递并且不会由一个位置参数来填充。keyword-only参数出现在`*args`参数之后，可以使用默认值。

```python
def kwonly(a, *b, c=0):
    print(a,b,c)
kwonly(1,2,3,c=4)  # c为keyword-only参数，必须使用关键字传递
```

在参数列表中可以使用一个`*`字符，表示函数不接受变长的参数列表，而且其后的参数都是关键字参数。

```python
def kwonly(a, *, b, c):
    print(a,b,c)
kwonly(1,b=3,c=4)  # b, c为keyword-only参数，必须使用关键字传递
```

## 第19章 函数的高级话题

函数设计包括功能分解（聚合）和通信（耦合），其原则：

- 耦合性：对于输入使用参数，对于输出使用return语句，力求使函数独立于外部的世界
- 耦合性：只有在真正必要的情况下使用全局变量
- 耦合性：不要改变可变类型的参数，除非调用者希望这样做
- 聚合性：每一个函数都应该有一个单一的、统一的目标。每个函数都应该做一件事，这件事可以用一个简单语句说明。
- 大小：每一个函数应该相对较小。
- 耦合：避免直接改变在另一个模块文件中的变量。

### 19.1 递归函数

递归函数是直接或间接地调用自身以进行循环的函数。递归函数可以用来遍历拥有任意的不可预知形状的结构。

线性的迭代应该适应简单的循环语句，但非线性迭代需要使用递归。如下面嵌套的列表求和：

```python
def sumtree(L):
    tot=0
    for x in L:
        if not isinstance(x,list)
        tot+=x
    else:
        tot+=sumtree(x)
    return tot
sumtree([1,[2,[3,[4,5]]]])
```

### 19.2 函数对象

函数是对象，适用对象的所有操作。函数对象嵌入其他数据结构中，可以作为参数传入函数，也可以作为函数的返回值。使用内省工具dir()检查函数对象的属性。

可以在函数对象上附加任意用户自定义属性，实现”静态本地变量”的效果（变量名对函数来说是本地的，但其值在函数退出后仍然保留）。属性与对象相关而不是与作用域相关。

```python
func.count=0
func.count+=1
```

函数注解是与函数参数和返回值相关的任意用户定义的数据，Python将注解收集在函数对象的`__annotations__`属性上，参数名和return作为键的字典中。参数注解在参数名称加冒号后，默认值前；返回值注解在参数列表加->后，冒号前。函数注解只在def语句中有效，在lambda表达式中无效。

```python
def func(a:'spam', b:(1,10)=0, c:float)->int:
    pass
func.__annotations__
```

### 19.3 匿名函数

lambda是一种生成函数对象的表达式，它返回一个函数而不是将这个函数赋值给一个变量名。其一般形式：

```python
lambda arg1, arg2,...argN: expression using args
```
与def语句定义函数相比，lambda的特点：

- lambda是一个表达式，而不是一个语句。lambda可以出现在def无法出现的地方，如列表和函数参数中；lambda返回一个值（函数对象），可以选择性的赋给一个变量名，而def总是将函数赋给变量名，而不是作为结果返回。
- lambda的主体是一个单个的表达式，而不是一个代码块。lambda为编写简单函数而设计

在lambda参数中可以使用默认参数。

lambda表达式引入的本地作用域更像一个嵌套的def语句，遵循LEGB作用域查找规则。

```python
def action(x):
    return lambda y:x+y
```

lambda常用在def语句不能使用的地方，也用在函数式编程工具中（对序列应用一些函数的工具）：

- map()，对一个序列对象中的每个元素应用传入的函数，返回一个可迭代对象。如：map(lambda x: x+3,      alist)
- filter() ，基于某一测试函数过滤出一些元素，返回一个可迭代对象。如：filter(lambda   x:x>0, range(-5,5))
- reduce()，在functools模块中，对每两个元素都应用函数并运行到最后结果。如：reduce(lambda  x,y:x+y, [1,2,3,4])

## 第20章 迭代器和解析，第二部分

### 20.1 列表解析

列表解析可以嵌套任意数量的for循环，并且每个循环都有可选的if测试，其通用的结构：

```python
[expression for target1 in iterable1 [if condition1]
            for target2 in iterable2 [if condition2] ... ]
```

### 20.2 生成器

Python提供了两种延迟创建结果的工具，即在需要的时候产生结果，而不是立即产生：

- 生成器函数：使用yield表达式一次返回一个结果，在每个结果之间挂起和继续它们的状态。
- 生成器表达式：创建一个生成器对象，语法和列表解析相同，只是在圆括号中。

生成器函数可以生成一个值，并随后从其退出的地方继续。生成器函数创建一个生成器对象，支持迭代协议。

生成器对象控制生成器函数的执行，当生成器对象的`__next__`方法被调用时生成器启动，执行到yield表达式，挂起函数，向调用者发送一个值；当生成器重启时，从上一个状态继续执行，直到执行完所有的yield表达式，引发一个StopIteration异常。

生成器函数也可以有一条return语句，总是在def语句块末尾，直接终止值的产生。

```python
def gen(N):
    for i in range(N):
        x = yield i**2  # yield为表达式，而不是语句
        print(x)
G=gen(5)  # 生成器函数gen(),创建生成器对象G
```

生成器的扩展：生成器的启动，可以调用`__next__`方法，yield表达式的值为None；也可以调用send(value)方法，yield表达式的值为value，借此实现调用者与生成器的通信。用send方法首次启动生成器时必须传入None。

迭代器和列表解析概念的结合就形成了生成器表达式，与列表解析不同的是在圆括号中，如果生成器表达式本身在括号内，则其括号可省略。

```python
(x**2 for x in range(5))
sum(x**2 for x in range(5))  # 生成器表达式本身的括号省略
```

生成器函数和生成器表达式都创建了一个迭代器对象，并且只支持单次活跃迭代。不能创建多个迭代器；当迭代器运行完成，必须创建一个新的迭代器才能再次开始。

Python的解析语法包括列表解析、集合解析、字典解析、生成器表达式。使用生成器表达式的解析语法：

- 列表解析，`[f(x) for x in S  if p(x)]`，等同于list(f(x) for x in  S if p(x))
- 集合解析，`{f(x) for x in S  if p(x)}`，等同于set(f(x) for x in  S if p(x))
- 字典解析，`{x:f(x) for x in  items  if p(x)}`，等同于dict((x,f(x)) for x in items  if p(x))

## 第21章 宏伟蓝图

模块是Python最高级别的程序组织单元，对应于程序文件，通过其命名空间将程序组织成系统。模块的作用：

* 代码重用
* 命名空间划分
* 服务和数据共享。

使用import语句第一次导入模块时，执行三个操作：

1. 找到模块文件

## 第26章 类代码编写基础

OOP模型的两种对象：

- 类对象，执行class语句产生，提供默认行为，是实例对象的工厂
  - class语句创建类对象并将其赋值给变量名
  - class语句中的赋值语句创建类的属性
  - 类属性提供对象的状态和行为
- 实例对象，是程序处理的实际对象，有独立的命名空间，继承了类中的变量。
  - 像函数那样调用类对象会创建新的实例对象
  - 每个实例对象继承类的属性并获得了自己的命名空间
  - 在方法内对self属性做赋值运算会产生每个实例自己的属性。self是类方法的第一个参数，引用正在处理的实例对象

程序可以创建、取出、修改其所引用的任何对象的属性，包括类对象和实例对象。

类的继承机制：

- 超类列在类开头的括号中，如，class sonclass(superclass1,  superclass2):
- 类从其超类中继承属性
- 实例会继承所有可读取类的属性
- 每个object.attribute都会开启新的独立搜索，这种属性搜索包括类外（object.att）和类内（self.data）
- 逻辑的修改是通过创建子类，而不是修改超类。子类中重新定义超类的属性，称为重载

运算符重载的主要概念：

- 以前后双下划线命名的方法（`__method__`）是特殊的钩子。Python提供特殊命名的方法来拦截运算符，为每种运算和特殊命名的方法之间定义了固定的映射关系
- 当实例出现在内置运算时，这类方法会自动调用
- 类可重载多数内置类型运算符
- 运算符重载方法没有默认值，而且也不需要。如没有重载，则类的实例不支持相应运算
- 运算符重载可让类与Python的对象模型相集成。提供了一致的接口

一个常见的运算符重载方法是`__init__`，也称构造函数，用于初始化对象的状态。`__init__`方法和`self`参数是了解Python OOP代码的关键之一。

Python的类模型是动态的。类和实例本身是一个对象，具有独立完备的命名空间。属性可以通过赋值语句动态建立。可以建立一个空类，然后通过赋值变量为类添加属性。同样，对实例对象的属性进行赋值，也会创建实例对象的属性。对象的方法也可以这样动态添加。

```python
class rec: pass
rec.name="John"
x=rec()  # 创建一个空的实例对象
x.name="Bob"  # 创建实例对象的属性，而不是修改类的属性
def func(self): pass
rec.method = func
```

特殊的类属性：

- 基于类的对象具有`__dict__`属性，包含对象命名空间的字典；
- 实例对象的`__class__`属性，记录实例对象继承的类；
- 类对象的`__bases__`属性，记录类对象的超类的元组。

## 第28章 类代码编写细节

### 28.1 类的定义

class与def类似，为可执行语句，隐含了赋值运算，其一般形式为：

```python
class name(superclass, ...):
    data=value
    def method(self, ...):
        self.member=value
```

类就是一个命名空间，对数据和逻辑进行封装，类的独特之处就在于其命名空间也是继承的基础：

* 与函数类似，class语句建立了本地作用域，其中的赋值语句建立本地作用中的变量
* 与模块类似，其中的赋值语句建立的变量成为类对象的属性。

类的数据属性为所有实例共享，用于管理贯穿所有实例的信息。对实例的属性进行赋值，会创建或修改实例的属性，而不是类的属性。对属性的搜索是在引用是发生，而不是赋值。对象属性的赋值总是修改对象，除法重载该类的`__setattr__`方法，重新定义了赋值运算。

类方法的第一个参数总是接收方法调用的隐性主体，即实例对象。Python自动把实例方法的调用对应到类方法，即将`instance.method(args...)`翻译为`class.method(instance,      args...)`。方法定义时使用一个位置参数，第一个参数按惯例为self，但只是其位置重要。

每个`obj.attr`表达式都会搜索类的属性树，直到找到一个attr。属性树的构造：

* 实例属性由方法内self属性赋值运算生成
* 类属性通过class语句内的赋值语句生成
* 超类的连接通过class语句首行的括号内列出的类生成

多重继承情况下，搜索属性时，会从左至右搜索类首行的超类，直到找到相符者。

### 28.1 抽象超类

抽象超类的部分方法没有定义，需要由子类来定义。如果在子类中没有定义，则会引发未定义变量名的异常。可以使用assert语句，raise语句或特殊语法定义：

```python
#1 使用assert语句

    def action(self):
        assert False, "Action must be defined"
        
#2 使用raise语句

    def action(self):
        raise NotImplementedError, "Action must be defined"
        
# 使用特殊语法

```

### 28.3 命名空间

点号对变量作用域不同处理：

* 无 点号运算的变量名与作用域相对应
* 点号的属性名使用对象的命名空间
* 有些作用域会对对象的命名空间进行初始化（模块和类）

简单变量，赋值使其成为本地变量，引用则遵循LEGB法则。

点号变量名为对象属性，赋值语句在对象的命名空间中创建或修改属性，模块属性的引用从对象中直接读取，类属性的引用搜索类的继承树。

**Python中赋值变量名的位置决定了变量所在的作用域和对象。**

Pthon的命名空间是以字典的形式实现的，并有内置属性`__dict__`读取。实例和类对象是带有链接的字典，对属性的点号运算就是字典的索引运算，属性的搜索就是搜索链接的字典。

类的继承树可以通过特殊属性查看：实例的`__class__`属性链接到它的类，类的`__bases__`属性是一个元组，链接到超类。

### 31.2 静态方法和类方法

Python 类支持三种方法：实例方法、静态方法、类方法：

* 实例方法，用实例对象调用
* 静态方法，是类中无self参数的简单方法，旨在操作类属性而不是实例属性，通常用于记录类跨实例的类信息。如果只通过类调用的话，无需将方法声明为静态的，如果要通过实例调用，则必须声明为静态的。如`mname=staticmethod(mname)`，或使用函数装饰器`@staticmethod`
* 类方法，传递给它的第一个参数是一个类而不是一个实例。必须声明为类方法，否则按实例方法处理。如`mname=classmethod(mname)`，或使用函数装饰器`@classmethod

```python
class Methods:
    def imeth(self, x):  # Normal
        print(self, x)

    @staticmethod
    def smeth(x):  # Static
        print(x)

    @classmethod
    def cmeth(cls, x):  # Class
        print(cls, x)
        
obj=Methods()

obj.imeth(1)
Methods.imeth(obj, 2)

Methods.smeth(3)
obj.smeth(4)

Methods.cmeth(5)
obj.cmeth(6)
```

异常是可以改变程序中控制流程的事件，根据错误自动触发，也可由代码触发和截获。由四个语句处理（第一种有两个变异，分别列出）：

- 事件通知，发出有效的状态信息，而不需要刻意传递结果标志，或刻意对其进行测试。
- 特殊情况处理，在异常处理器中处理某些罕见的情况，省去编写特殊情况处理代码。

- 终止行为，try/final语句无论是否有异常，一定会进行结束运算。

## 第33章 异常编码细节

### 23.1 try语句

try/except/else语句，包含一个或多个except分句来识别要捕捉的异常，一个可选的else分句（没有异常时执行）和finally分句（无论有无异常都执行）：

```python
try:  statements
except (type1, type2, …) as name: statements
else:  statements
finally: statements
```

try语句是一个程序环境，try下面的代码块为程序的主要功能，except子句为异常处理器，else子句为无异常处理器，try语句的形式：

* except:  捕捉所有（其他）异常类型
* except name:  只捕捉特定的异常
* except name, value:  捕捉所列的异常和其额外的数据（或实例）
* except (name1, name2):  捕捉任何列出的异常
* except (name1, name2), value:  捕捉任何列出的异常，并取得其额外的数据
* else:  如果没有引发异常，就运行
* final:  总是会运行此代码块

空的exept作为一个通用的功能，会捕捉所有的异常，包括不希望捕捉的异常；捕捉一个名为Exception的异常几乎与一个空的exept具有相同的效果，但忽略和系统退出相关的异常。

except分句带有as name时，name会被赋值引发的异常实例，将使异常处理器能够访问实例的数据和异常类的方法。

try/finally语句，无论是否发生异常，finally分句一定执行，用于清理动作，如关闭文件、断开服务等。finally不会终止异常，如果有未处理异常，之后还会继续传递。

```python
try:  statements
finally: statements
```

### 23.2 raise语句

raise语句用于显式的触发异常。后面是可选的要引发的类或类的一个实例（异常总是类的实例），如果提供类，Python调用不带构造函数参数的类，创建被引发的一个实例：

```
raise instance
raise class # 等价于：raise class()
raise
```

不带实例的raise语句重新引发最近的异常。异常一旦被捕获，就会死掉，如果想要捕获并处理一个异常，又不希望异常死掉，就使用不带实例的raise语句。

当使用from时，其指定的异常会附加到引发异常的`__cause__`属性。类似的，当异常处理器内引发异常时，前一个异常会附加到新的异常的`__context__`属性。

```python
raise exception from otherexception
```

### 23.3 assert语句

assert语句，可视为条件式的raise语句，如果test为假，就引发异常AssertionErro(data)，data可选，为异常提供额外信息：

```python
assert test, data
```

assert语句总是用来收集用户定义的约束条件，而不是捕捉内在的程序设计错误。

### 23.4 with环境管理器

with/as环境管理器，可作为try/finally模式的替代，不过该语句支持基于对象的环境管理协议：

```python
with expression [as  varible]:
    statements
```

其中，expression返回一个对象，支持环境管理协议，如果有as子句，此对象的入口方法的返回值会赋给变量名varible。

with语句实现环境管理协议的方式：

1. 计算表达式，所得到的对象称为环境管理器，它必须有`__enter__`和`__exit__`方法；
2. 环境管理器的`__enter__`方法被调用，如果as子句存在，其返回值赋值给as子句的变量，否则丢弃；
3. 执行with代码块；
4. 如果引发异常，`__exit__(type, value, traceback)`方法被调用（这也是sys.exc_info返回的相同值），如果此方法返回值为假，则异常会重新引发，否则异常会终止；
5. 如果没有异常，`__exit__(None,      None, None)`方法被调用。

## 第34章 异常对象

异常都是通过类实例对象来表示的。类可以组织异常分类，可以附加异常的信息，而且支持继承。except子句使用超类关系进行匹配，即except可以捕捉该类及其全部子类的实例。

Python把内置异常组织成层次结构：

- LookupError  使用键或索引查询引发的异常的超类，派生：IndexError,      KeyError


## 第35章 异常的设计

嵌套异常处理器：

Python在运行时将try语句的进入点压栈，发生异常时，会匹配except子句，回到最近的try语句，实现异常向上传递至较高的处理器。

* 关闭文件和服务器连接，更简洁的方法是使用环境管理器
* 进行程序调试，在外层try中包含程序，这样就可以屏蔽默认异常处理器终止程序

作为函数信号的一般用法：

```python
class Failure(Exception): pass
def searcher():
    if success:
        return finditem

try:

except Failure:
    report
else:
    process item
```

使用异常处理器取代默认的异常处理，从sys.exc_info()取得异常信息，使程序保持运行状态。


在进程中循环测试的一般用法：


异常的使用技巧：

* 将经常出错的代码包含在try语句中，如文件操作、网络接口调用等，除非希望使用Python默认的异常处理

### 36.1 字符串基础知识

字符编码方法：

- Unicode，又叫宽字符，每个字符可能表示为多个字节。

Python包含100多种编码，可查阅帮助文档或导入encodings模块，使用help(encodings)查看。

### 36.2 Python3的字符串类型

Python3有3种字符串类型，一种用于文本数据，两种用于二进制数据：

- bytes  表示二进制数据，定义为一个8位整数的不可变序列，表示绝对的字节值，支持str所有的操作，不包括字符串格式化
- bytearray  是bytes类型的一个变体，一种可变的bytes类型并支持原处修改，支持str和bytes常见的字符串操作，并支持列表的原处修改操作

### 36.3 文本和二进制文件

文件I/O的区别：

- 文本文件，以文本模式打开，读取时自动解码（使用平台默认的或指定的编码），返回str；写入时，接受str，并自动编码，对应常量：'...',  '''…'''
- 二进制文件，以二进制模式打开，读写时不做任何修改（编解码，行末转换），接受bytes对象，写入时也接受bytearray对象，对应常量：b'…'

文本模型的文件执行通用的行末转换：默认地，所有的行末形式映射为脚本中的一个单个的'\n'字符，输出文本文件时把'\n'行末符号转换为'\r\n'，在输入时，把'\r\n'转换回'\n'，而不管在什么平台上运行。文本文件也负责阅读和写入在某些Unicode编码方案中存储文件开始处的字节顺序标记（Byte Order Mark，BOM）。

由于二者的显著差异，所以必须确定数据是文本或二进制，在脚本中相应地使用str或bytes对象来处理，open()的模式字符串参数至关重要。

如果试图向一个文本文件写入一个bytes，或者向二进制文件写入一个str，将会得到错误。

读写Unicode文件的时候，可以指定编码格式，Python自动完成数据的编解码，如，open('file', 'r', encoding='utf-8')。

某些编码方案在文件开始处存储字节顺序标记（Byte Order Mark, BOM），来指定数据的大小尾方式或声明编码类型。如果编码名暗示了BOM，Python文件读写会忽略该标志，但有时需要指定编码名称来显式地处理BOM。例如utf-8会忽略BOM，而utf-8-sig会显式处理BOM。

## 第37章 管理属性

属性管理常用的工具：

- `__getattr__`和`__setattr__`方法，把未定义的属性的获取和所有的属性的赋值指向通用的处理器方法
- `__getattribute__`方法，把所有属性的获取都指向所有类中的一个泛型处理器方法
- property内置函数，把特定属性的访问定位到get和set处理器函数， 也叫作特性（property）。拦截属性删除，提供属性文档
- 描述符协议，把特定属性的访问定位到具有任意get和set处理器方法的类的实例

后两者技术适用于特定属性，而前两种技术更通用。

### 37.1 特性

特性协议允许我们把一个特定属性的get和set操作指向我们所提供的函数或方法，使得我们能够插入在属性访问的时候自动运行的代码，拦截属性删除，并且如果愿意的话，还可为属性提供文档。

通过内置函数property创建特性并将其赋值给属性，它的访问拦截功能通过self实例参数提供。一个特性管理一个单个的、特定的属性。

通过把内置函数property的结果赋给一个类属性来创建一个特性：

```attribute=property(fget, fset, fdel, doc) ```

注：所有参数都有默认值None，忽略set函数使属性成为只读属性。

向fget传递一个函数来拦截属性访问，给fset传递一个函数进行赋值，并且给fdel传递一个函数进行属性删除；doc参数接收该属性的一个文档字符串，如果想要的话（否则，该特性会赋值fget的文档字符串，如果提供了fget的文档字符串的话，其默认值为None）。

fget返回计算过的属性值，并且fset和fdel不返回什么（确实是None）。

这个内置的函数调用返回一个特性对象，我们将它赋给了在类的作用域中要管理的属性的名称。

特性的示例：如下的类使用一个特性来记录对一个名为name的属性的访问，实际存储的数据名为_name，以便不会和特性搞混了：

```python
class Person:
    def __init__(self, name):
        self._name = name
    def getName(self):
        print('fetch...')
        return self._name
    def setName(self, value):
        print('change...')
        self._name = value
    def delName(self):
        print('remove...')
        del self._name
    name = property(getName, setName, delName, "name property docs")
    
bob = Person('Bob Smith')     # bob has a managed attribute
print(bob.name)       # Runs getName
bob.name = 'Robert Smith'     # Runs setName
print(bob.name)
del bob.name       # Runs delName

# 实例和较低的子类都继承特性，如下得到同样的结果
class Super:
    ...the original Person class code...
    name = property(getName, setName, delName, 'name property docs')

class Person(Super):
    pass         # Properties are inherited

bob = Person('Bob Smith')
...rest unchanged...
```

可以使用装饰器编写特性。内置函数property可以充当一个装饰器，来定义一个函数，当获取一个属性的时候自动运行该函数：

```python
class Person:
    @property
    def name(self): ...     # Rebinds: name = property(name)
```

运行的时候，装饰的方法自动传递给property内置函数的第一个参数。这其实只是创建一个特性并手动绑定属性名的一种替代语法：

```python
class Person:
    def name(self): ...
    name = property(name)
```

property对象也有getter、setter和deleter方法，这些方法指定相应的特性访问器。可以使用这些方法，通过
装饰常规方法来指定特性的组成部分，其中getter部分通常由创建特性自身的行为自动填充：

```python
class Person:
    def __init__(self, name):
    self._name = name

    @property
    def name(self):    # name = property(name)
        "name property docs"
        print('fetch...')
        return self._name

    @name.setter
    def name(self, value):     # name = name.setter(name)
    print('change...')
    self._name = value

    @name.deleter
    def name(self):      # name = name.deleter(name)
    print('remove...')
    del self._name

bob = Person('Bob Smith')     # bob has a managed attribute
print(bob.name)       # Runs name getter (name 1)
bob.name = 'Robert Smith'     # Runs name setter (name 2)
print(bob.name)
del bob.name       # Runs name deleter (name 3)
```

### 37.2 描述符

描述符提供了拦截属性访问的一种替代方法；实际上，特性是描述符的一种。从技术上讲，property内置函数只是创建一个特定类型的描述符的一种简化方式，而这种描述符在属性访问时运行方法函数。

从功能上讲，描述符协议允许把一个特定属性的get和set操作指向我们提供的一个单独类对象的方法：它们提供了一种方式来插入在访问属性的时候自动运行的代码，并且它们允许我们拦截属性删除并且为属性提供文档（如果愿意的话）。

描述符作为单独的类编写，并且它们就像方法函数一样分配给类属性，可以通过子类和实例继承。通过为描述符自身提供一个self，以及提供客户类的实例，都可以提供访问拦截方法。

和特性一样，描述符也管理一个单个的、特定的属性。特性实际上只是创建一种特定描述符的方便方法，它们可以直接作为描述符编写。然而，特性的应用领域相对狭窄，描述符提供了一种更为通用的解决方案。

描述符作为单独的类编写，并且针对想要拦截的属性访问操作提供特定命名的访问器方法，将其实例赋值给类属性，通过其自身的self，以及客户类的实例，实现访问拦截。当以相应的方式访问分配给描述符类实例的属性时，描述符类中的获取、设置和删除等方法自动运行：

```python
class Descriptor:
    "docstring goes here"
    def __get__(self, instance, owner): ...   # Return attr value
    def __set__(self, instance, value): ...   # Return nothing (None)
    def __delete__(self, instance): ...   # Return nothing (None)
```

带有任何这些方法的类都可以看作是描述符，并且当它们的一个实例分配给另一个类的属性的时候，它们的这些方法是特殊的，当访问属性的时候，会自动调用它们。

如果这些方法中的任何一个空缺，通常意味着不支持相应类型的访问。然而，和特性不同，省略一个`__set__`意味着允许这个名字在一个实例中重新定义，因此，隐藏了描述符。要使得一个属性是只读的，我们必须定义`__set__`来捕获赋值并引发一个异常。

要使基于描述符的属性成为只读属性，需要捕获赋值并引发一个异常来阻止，如，

```def __set__(*args): raise AttributeError('cannot set')```

所有3种描述符方法，都传递了描述符类实例（self）以及描述符实例所附加的客户类的实例（instance），`__get__`访问方法还额外地接收一个owner参数，指定了描述符实例要附加到的类。

其instance参数要么是访问的属性所属的实例（用于instance.attr），要么当所访问的属性直接属于类的时候是None（用于class.attr）。前者通常针对实例访问计算一个值；如果描述符对象访问是受支持的，后者通常返回self。

```python
class Descriptor():
    def __get__(self, instance, owner):
        print(self, instance, owner, sep='\n')
    def __set__(*args):
        raise AttributeError('cannot set')  # Make attribute read-only

class Subject:
    attr = Descriptor()   # Descriptor instance is class attr

X = Subject()
X.attr
Subject.attr
```

用描述符实现前面的例子：

```python
class Name:
    "name descriptor docs"
    def __get__(self, instance, owner):
        print('fetch...')
        return instance._name
    def __set__(self, instance, value):
        print('change...')
        instance._name = value
    def __delete__(self, instance):
        print('remove...')
        del instance._name

class Person:
    def __init__(self, name):
        self._name = name
    name = Name()   # Assign descriptor to attr

bob = Person('Bob Smith')  # bob has a managed attribute
print(bob.name)   # Runs Name.__get__
bob.name = 'Robert Smith'   # Runs Name.__set__
print(bob.name)
del bob.name   # Runs Name.__delete__

# 描述符类实例是一个类属性，因此由客户类和任何子类的所有实例所继承
class Super:
    def __init__(self, name):
        self._name = name
    name = Name()
class Person(Super):      # Descriptors are inherited
    pass
```

描述符可以使用实例状态和描述符状态，或者二者的任何组合：

* 描述符状态用来管理内部用于描述符工作的数据。
* 实例状态记录了和客户类相关的信息，以及可能由客户类创建的信息。

### 37.3 `__getattr__`和`__getattribute__`方法

`__getattr__`和`__getattribute__`操作符重载方法提供了拦截类实例的属性获取的另一种方法。就像特性和描述符一样，它们也允许我们插入当访问属性的时候自动运行的代码。

通用的属性访问拦截的方法：

* `__getattr__`，针对未定义的属性运行，即属性没有存储在实例上，或没有从其类继承

- `__getattribute__`，针对每个属性，使用时必须小心避免通过把属性访问传递给超类而导致递归循环

- `__setattr__`，捕获赋值对属性的更改，针对每个属性，必须小心避免由于通过实例命名空间字典指向属性赋值而导致的递归循环
- `__delattr__`，拦截属性删除，必须以同样的方式避免循环

与特性和描述符不同，这些方法是Python的操作符重载协议的一部分，是类的特殊命名的方法，由子类继承，并且当在隐式的内置操作中使用实例的时候自动调用。

这些方法比特性和描述符更加通用，它们可以用来拦截对任何（几乎所有的）实例属性的访问，而不仅仅只是分配给它们的那些特定名称。

如果一个类定义了或继承了如下方法，那么当一个实例用于后面的注释所提到的情况时，它们将自动运行：

```python
def __getattr__(self, name):    # On undefined attribute fetch [obj.name]
def __getattribute__(self, name):   # On all attribute fetch [obj.name]
def __setattr__(self, name, value):   # On all attribute assignment [obj.name=value]
def __delattr__(self, name):    # On all attribute deletion [del obj.name]
```

其中，self通常是主体实例对象，name是将要访问的属性的字符串名，value是要赋给该属性的对象。两个get方法通常返回一个属性的值，另两个方法不返回什么（None）。

由于`__getattribute__`和`__setattr__`针对所有的属性运行，其代码要注意在访问其他属性的时候避免再次调用自己并触发一次递归循环。

在一个`__getattribute__`方法代码内部的另一次属性获取，将会再次触发`__getattribute__`，并且代码将会循环直到内存耗尽：

```python
def __getattribute__(self, name):
    x = self.other         # LOOPS!
```

要解决这个问题，把获取指向一个更高的超类，而不是跳过这个层级的版本。object类总是一个超类，并且它在这里可以很好地起作用：

```python
def __getattribute__(self, name):
    x = object.__getattribute__(self, 'other')    # Force higher to avoid me
```

对于`__setattr__`，情况是类似的。在这个方法内赋值任何属性，都会再次触发`__setattr__`并创建一个类似的循环：

```python
def __setattr__(self, name, value):
    self.other = value         # LOOPS!
```

要解决这个问题，把属性作为实例的`__dict__`命名空间字典中的一个键赋值。这样就避免了直接的属性赋值：

```python
def __setattr__(self, name, value):
    self.__dict__['other'] = value       # Use atttr dict to avoid me
```

尽管这种方法比较少用到，但`__setattr__`也可以把自己的属性赋值传递给一个更高的超类而避免循环，就像`__getattribute__`一样：

```python
def __setattr__(self, name, value):
    object.__setattr__(self, 'other', value)     # Force higher to avoid me
```

相反，我们不能使用`__dict__`技巧在`__getattribute__`中避免循环。获取`__dict__`属性本身会再次触发`__getattribute__`，导致一个递归循环。

`__delattr__`方法实际中很少用到，但是，当用到的时候，它针对每次属性删除而调用。因此，我们必须小心，在删除属性的时候要避免循环，通过使用同样的技术：命名空间字典或者超类方法调用。

使用操作符重载机制重写上一个实例：

```python
class Person:
    def __init__(self, name):      # On [Person()]
        self._name = name       # Triggers __setattr__!

    def __getattr__(self, attr):     # On [obj.undefined]
        if attr == 'name':       # Intercept name: not stored
            print('getattr fetch...')
            return self._name      # Does not loop: real attr
        else:         # Others are errors
            raise AttributeError(attr)
            
    # Replace __getattr__ with this
    def __getattribute__(self, attr):      # On [obj.any]
        if attr == 'name':         # Intercept all names
            print('getattribute fetch...')
            attr = '_name'         # Map to internal name
        return object.__getattribute__(self, attr)   # Avoid looping here
    
    def __setattr__(self, attr, value):   # On [obj.any = value]
        if attr == 'name':
            print('change...')
            attr = '_name'       # Set internal name
        self.__dict__[attr] = value     # Avoid looping here
        
    def __delattr__(self, attr):     # On [del obj.any]
        if attr == 'name':
            print('remove...')
            attr = '_name'       # Avoid looping here too
        del self.__dict__[attr]     # but much less common
        
bob = Person('Bob Smith')       # bob has a managed attribute
print(bob.name)         # Runs __getattr__
bob.name = 'Robert Smith'       # Runs __setattr__
print(bob.name)
del bob.name         # Runs __delattr__
```

注意，`__init__`构造函数中的属性赋值也触发了`__setattr__`，这个方法捕获了每次属性赋值，即便是类自身之中的那些。

`__getattr__`和`__getattribute__`方法之间的区别：`__getattr__`拦截对未定义属性的获取；`__getattribute__`拦截所有的属性获取，并且必须将那些没有管理的属性访问指向超类获取器以避免循环。尽管`__getattribute__`可以捕获比`__getattr__`更多的属性获取，但是实际上，它们只是一个主题的不同变体。如果属性没有物理地存储，二者具有相同的效果。

## 第38章 装饰器

### 38.1 装饰器概念

装饰是为函数和类指定管理代码的一种方式。装饰器本身的形式是处理其他的可调用对象的可调用的对象（如函数）。有两种形式：

- 函数装饰器，在函数定义的时候进行名称重绑定，提供一个逻辑层来管理函数和方法，或随后对它们的调用
- 类装饰器，在类定义的时候进行名称重绑定，提供一个逻辑层来管理类，或随后调用它们所创建的实例

简而言之，装饰器提供了一种方法，在函数和类定义语句的末尾插入自动运行代码。

装饰器的用途：

* 管理调用和实例：通过安装包装器对象，在需要的时候拦截随后的函数调用和类实例创建，并处理它们。
* 管理函数和类：直接管理函数对象和类对象。

### 38.2 装饰器基础

#### 38.2.1 函数装饰器

函数装饰器，为函数明确了特定的运算模式，也就是将函数包裹了另一层，在另一函数的逻辑内实现。它是一种语法糖：通过在一个函数的def语句的末尾来运行另一个函数，把最初的函数名重新绑定到结果。

函数装饰器是一种关于函数的运行时声明，函数的定义需要遵守此声明。其语法是在def语句的前一行，使用@加元函数，也就是管理另一函数（或其他可调用对象）的函数。

函数装饰器自动将如下的代码：

```python
@decorator     # Decorate function
def F(arg):
    ...
F(99)       # Call function
```

映射为这一对等的形式，其中装饰器是一个单参数的可调用对象，它返回与F具有相同数目的参数的一个可调用对象：

```python
def F(arg):
    ...
F = decorator(F)   # Rebind function name to decorator result
F(99)       # Essentially calls decorator(F)(99)
```

当随后调用F函数的时候，它自动调用装饰器所返回的对象，该对象可能是实现了所需的包装逻辑的另一个对象，或者是最初的函数本身。

装饰器自身是一个返回可调用对象的可调用对象。当随后装饰的函数通过其最初的名称调用的时候，将会调用这个对象。

要在一个函数创建之后接入装饰协议以管理函数，编写如下形式的装饰器：

```python
def decorator(F):
	# Process function F
	return F
@decorator
def func(): ...     # func = decorator(func)
```

更典型的用法，是插入逻辑以拦截对函数的随后调用，我们可以编写一个装饰器来返回和最初函数不同的一个对象：

```python
def decorator(F):
	# Save or use function F
	# Return a different callable: nested def, class with __call__, etc.
@decorator
def func(): ...     # func = decorator(func)
```

更概括地说，装饰器返回了一个包装器，包装器把最初的函数保持到一个封闭的作用域中：

```python
def decorator(F):       # On @ decoration
	def wrapper(*args):     # On wrapped function call
		# Use F and args
		# F(*args) calls original function
	return wrapper
@decorator         # func = decorator(func)
def func(x, y):       # func is passed to decorator's F
```

基于类的装饰器，需要重载调用操作`__call__`方法，并且使用实例属性而不是封闭的作用域：

```python
class decorator:
	def __init__(self, func):    # On @ decoration
		self.func = func
	def __call__(self, *args):   # On wrapped function call
		# Use self.func and args
		# self.func(*args) calls original function
@decorator
def func(x, y):       # func = decorator(func)
```

上述编码对于拦截简单函数调用有效，但当它应用于类方法函数的时候，并不是很有效。当装饰器的`__call__`方法随后运行的时候，其中的self接收装饰器类实例，并且被装饰类的实例不会包含到一个`*args`中。为了支持函数和方法，嵌套函数的替代方法工作得更好：

```python
def decorator(F):       # F is func or method without instance
	def wrapper(*args):     # class instance in args[0] for method
		# F(*args) runs func or method
	return wrapper

@decorator
def func(x, y):       # func = decorator(func)
	...
func(6, 7)         # Really calls wrapper(6, 7)

class C:
	@decorator
	def method(self, x, y):     # method = decorator(method)
		...        # Rebound to simple function
X = C()
X.method(6, 7)       # Really calls wrapper(X, 6, 7)
```

#### 38.2.2 类装饰器

类装饰器类似与函数装饰器，把一个类名重新绑定到一个可调用对象，可以用来管理类或创建实例的时候插入一个包装逻辑层来管理实例。类装饰器是管理类的一种方式，或者用管理或扩展类所创建的实例的额外逻辑，来包装实例构建调用。

```python
@decorator       # Decorate class
class C:
    ...
x = C(99)       # Make an instance
# 相当于以下代码：
class C:
    ...
C = decorator(C)     # Rebind class name to decorator result
x = C(99)       # Essentially calls decorator(C)(99)
```

类装饰器与函数装饰器使用的众多相同的技术来编码，类装饰器也是返回一个可调用对象的一个可调用对象。

```python
def decorator(C):
	# Process class C
	return C
@decorator
class C: ...         # C = decorator(C)
```

可以插入一个包装器层来拦截随后的实例创建调用，而是返回一个不同的可调用对象：

```python
def decorator(C):
	# Save or use class C
	# Return a different callable: nested def, class with __call__, etc.
@decorator
class C: ...         # C = decorator(C)
```

#### 38.2.3 装饰器嵌套

装饰器可以嵌套，以实现多层处理逻辑。为了支持多步骤的扩展，装饰器语法允许我们向一个装饰的函数或方法添加包装器逻辑的多个层。语法上，每个装饰器必须出现在自己的一行中。

```python
@A
@B
@C
def f(...):
	...
# 如下这样运行：
def f(...):
	...
f = A(B(C(f)))
```

每个装饰器处理前一个的结果，这可能是最初的函数或一个插入的包装器。

#### 38.2.4 装饰器参数

装饰器可以接受参数，这些参数传递给了真正返回装饰器的一个可调用对象，而装饰器反过来又返回一个可调用对象。

```python
@decorator(A, B)
def F(arg):
	...
F(99)
# 自动地映射到其对等的形式
def F(arg):
	...
F = decorator(A, B)(F)  # Rebind F to result of decorator's return value
F(99)         # Essentially calls decorator(A, B)(F)(99)
```

装饰器参数往往意味着可调用对象的3个层级：接受装饰器参数的一个可调用对象，它返回一个可调用对象以作为装饰器，该装饰器返回一个可调用对象来处理对最初的函数或类的调用。这3个层级的每一个都可能是一个函数或类，并且可能以作用域或类属性的形式保存了状态。

## 第39章 元类

### 39.1 元类概念

从某种意义上讲，元类只是扩展了装饰器的代码插入模式。元类允许我们拦截并扩展类创建，它们提供了一个API以插入在一条class语句结束时运行的额外逻辑。

元类主要是针对那些构建API和工具供他人使用的程序员。

Python的高级编程工具：

* 内省属性，像`__class__`和`__dict__`这样的特殊属性允许我们查看Python对象的内部实现方面，
  以便更广泛地处理它们。
* 运算符重载方法，像`__str__`和`__add__`这样特殊命名的方法，在类中编写来拦截并提供应用于类实例
  的内置操作的行为。
* 属性拦截方法，一类特殊的运算符重载方法提供了一种方法在实例上广泛地拦截属性访问：`__getattr__`、`__setattr__`和`__getattribute__`允许包装的类插入自动运行的代码，这些代码可以验证属性请求并且将它们委托给嵌入的对象。
* 类特性，内置函数property允许我们把代码和特殊的类属性关联起来，当获取、赋值或删除该属性的时候就自动运行代码。
* 类属性描述符，其实，特性只是定义根据访问自动运行函数的属性描述符的一种简洁方式。描述符允许我们在单独的类中编写`__get__`、`__set__`和`__delete__`处理程序方法，当分配给该类的一个实例的属性被访问的时候自动运行它们。
* 函数和类装饰器，装饰器的特殊的@可调用语法，允许我们添加当调用一个函数或创建一个类实例的时候自动运行的逻辑。
* 元类，它们允许我们在一条class语句的末尾，插入当创建一个类对象的时候自动运行的逻辑。这个逻辑不会把类名重新绑定到一个装饰器可调用对象，而是把类自身的创建指向特定的逻辑。

和类装饰器不同，装饰器通常是添加实例创建时运行的逻辑，元类在类创建时运行。同样的，它们都是通常用来管理或扩展类的钩子，而不是管理其实例。

### 39.2 元类模型

所有类都是Object类的子类，所有类都是type类的实例，元类是type类的子类。

Python的类模型：

* 通过类创建实例，类的实例有一些自己的状态信息属性，它们也从所创建自的类继承了行为属性。
* 类对象是名为type的对象的实例，type本身是一个类。

Python的类和类型的关系：实例创建自类，而类创建自type。Python 3.0中，“类型”的概念与“类”的概念合并了。这两者基本上是同义词，类是类型，类型也是类。即：

* 类型由派生自type的类定义。
* 用户定义的类是类型类的实例。
* 用户定义的类是产生它们自己的实例的类型。

本质上，类根本不是一个独立的概念：它们就是用户定义的类型，并且type自身也是由一个类定义的。

由于类实际上是type类的实例，从type的定制的子类创建类允许我们实现各种定制的类。

* type是产生用户定义的类的一个类。
* 元类是type类的一个子类。
* 类对象是type类的一个实例，或一个子类。
* 实例对象产生自一个类。

换句话说，为了控制创建类以及扩展其行为的方式，需要做的只是指定一个用户定义的类创建自一个用户定义的元类，而不是常规的type类。

注意，这个类型实例关系与继承并不完全相同：用户定义的类可能也拥有超类，它们及其实例从那里继承属性（继承超类在class语句的圆括号中列出，并且出现在一个类的`__bases__`元组中）。

Class语句协议：在一条class语句的末尾，并且在运行了一个命名空间词典中的所有嵌套代码之后，它调用type对象来创建class对象。

```python
class = type(classname, superclasses, attributedict)
```

type对象反过来定义了一个`__call__`运算符重载方法，当调用type对象的时候，该方法运行两个其他的方法：

```python
type.__new__(typeclass, classname, superclasses, attributedict)
type.__init__(class, classname, superclasses, attributedict)
```

`其中：__new__`方法创建并返回了新的class对象，并且随后`__init__`方法初始化了新创建的对象。这是type的元类子类通常用来定制类的钩子。

### 39.3 声明元类

类默认是type类创建的。要告诉Python用一个定制的元类来创建一个类，直接声明一个元类来拦截常规的类创建调用。在类标题中把想要的元类作为一个关键字参数列出来，继承超类也可以列在标题中，在元类之前。

```python
class Spam(Eggs, metaclass=Meta):
```

当以这些方式声明的时候，创建类对象的调用在class语句的底部运行，修改为调用元类而不是默认的type：

```python
class = Meta(classname, superclasses, attributedict)
```

由于元类是type的一个子类，所以type类的`__call__`把创建和初始化新的类对象的调用委托给元类，如果它定义了这些方法的定制版本：

```python
Meta.__new__(Meta, classname, superclasses, attributedict)
Meta.__init__(class, classname, superclasses, attributedict)
```

### 39.4 编写元类

元类是子类化了type对象并且拦截类创建调用的类。即把一个类对象的创建导向顶级type类的一个子类。元类通常重新定义type类的`__new__`或`__init__`方法，以实现对新的类对象的创建和初始化的控制。

最简单元类只是带有一个`__new__`方法的type的子类，该方法通过运行type中的默认版本来创建类对象：

```python
class Meta(type):
	def __new__(meta, classname, supers, classdict):
		# Run by inherited type.__call__
		return type.__new__(meta, classname, supers, classdict)
```

元类也可以接入`__init__`协议，由type对象的`__call__`调用：通常，`__new__`创建并返回了类对象，`__init__`初始化了已经创建的类。例如：

```python
class MetaOne(type):
	def __new__(meta, classname, supers, classdict):
		print('In MetaOne.new: ', classname, supers, classdict, sep='\n...')
		return type.__new__(meta, classname, supers, classdict)
	def __init__(Class, classname, supers, classdict):
		print('In MetaOne init:', classname, supers, classdict, sep='\n...')
		print('...init class object:', list(Class.__dict__.keys()))
class Eggs:
	pass

print('making class')
class Spam(Eggs, metaclass=MetaOne):  # Inherits from Eggs, instance of Meta
	data = 1       # Class data attribute
	def meth(self, arg):     # Class method attribute
		pass
print('making instance')
X = Spam()
print('data:', X.data)
```

尽管重新定义type超类的`__new__`和`__init__`方法是元类向类对象创建过程插入逻辑的最常见方法，其他的方案也是可能的。

#### 39.4.1 使用简单的工厂函数

元类根本不是真的需要类。class语句发布了一条简单的调用，在其处理的最后创建了一个类。因此，实际上任何可调用对象都可以用作一个元类，只要它接收传递的参数并且返回与目标类兼容的一个对象。实际上，一个简单的对象工厂函数就像一个类一样工作：

```python
def MetaFunc(classname, supers, classdict):
	print('In MetaFunc: ', classname, supers, classdict, sep='\n...')
	return type(classname, supers, classdict)
class Eggs:
	pass
print('making class')
class Spam(Eggs, metaclass=MetaFunc):    # Run simple function at end
	data = 1         # Function returns class
	def meth(self, args):
		pass
```

#### 39.4.2 用元类重载类创建调用

由于它们涉及常规的OOP机制，所以对于元类来说，也可能直接在一条class语句的末尾捕获创建调用，通过定义type对象的`__call__`。

```python
class SuperMeta(type):
	def __call__(meta, classname, supers, classdict):
		print('In SuperMeta.call: ', classname, supers, classdict, sep='\n...')
		return type.__call__(meta, classname, supers, classdict)
class SubMeta(type, metaclass=SuperMeta):
	def __new__(meta, classname, supers, classdict):
		print('In SubMeta.new: ', classname, supers, classdict, sep='\n...')
		return type.__new__(meta, classname, supers, classdict)
	def __init__(Class, classname, supers, classdict):
		print('In SubMeta init:', classname, supers, classdict, sep='\n...')
		print('...init class object:', list(Class.__dict__.keys()))
class Eggs:
	pass
print('making class')
class Spam(Eggs, metaclass=SubMeta):
	data = 1
	def meth(self, arg):
		pass
```

当这段代码运行的时候，所有3个重新定义的方法都依次运行。这基本上就是type对象默认做的事情。

#### 39.4.3 用常规类重载类创建调用

要使用常规的基于继承的名称查找，我们可以用常规类和实例实现同样的效果。但是注意，`__new__`和`__init__`在这里必须有不同的名称，否则，当创建SubMeta实例的时候它们会运行，而不是随后作为一个元类调用：

```python
class SuperMeta:
	def __call__(self, classname, supers, classdict):
		print('In SuperMeta.call: ', classname, supers, classdict, sep='\n...')
		Class = self.__New__(classname, supers, classdict)
		self.__Init__(Class, classname, supers, classdict)
		return Class
class SubMeta(SuperMeta):
	def __New__(self, classname, supers, classdict):
		print('In SubMeta.new: ', classname, supers, classdict, sep='\n...')
		return type(classname, supers, classdict)
	def __Init__(self, Class, classname, supers, classdict):
		print('In SubMeta init:', classname, supers, classdict, sep='\n...')
		print('...init class object:', list(Class.__dict__.keys()))
class Eggs:
	pass
print('making class')
class Spam(Eggs, metaclass=SubMeta()):   # Meta is normal class instance
	data = 1         # Called at end of statement
	def meth(self, arg):
		pass
```

### 39.5 实例与继承的关系

由于元类以类似于继承超类的方式来制定，因此它们乍看上去有点容易令人混淆。一些关键点有助于概括和澄清这一模型：

* 元类继承自type类。它们可以重新定义type对象的方法，需要的时候重载或定制它们。元类通常重新定义type类的`__new__`和`__init__`，以定制类创建和初始化，如果它们希望直接捕获类末尾的创建调用的话，它们也可以重新定义`__call__`。尽管不常见，它们甚至是返回任意对象而不是type子类的简单函数。
* 元类声明由子类继承。在用户定义的类中，`metaclass=M`声明由该类的子类继承，因此，对于在超类链中继承了这一声明的每个类的构建，该元类都将运行。
* 元类属性没有由类实例继承。元类声明指定了一个实例关系，它和继承不同。由于类是元类的实例，所以元类中定义的行为应用于类，而不是类随后的实例。实例从它们的类和超类获取行为，但是，不是从任何元类获取行为。从技术上讲，实例属性查找通常只是搜索实例及其所有类的__dict__字典；元类不包含在实例查找中。

### 39.6 元类与类装饰器的关系

类装饰器和元类在功能上有重合。这源自于如下的事实：

* 在class语句的末尾，类装饰器把类名重新绑定到一个函数的结果。
* 元类通过在一条class语句的末尾把类对象创建过程路由到一个对象来工作。

这些略有不同的模型通常可实现同样的目标，虽然采用的方式不同。实际上，类装饰器可以用来管理一个类的实例以及类自身。尽管装饰器可以自然地管理类；然而，用元类管理实例有些不那么直接，元类可能最好用于类对象管理。

简单是最好的工程，高级工具应该只在需要的时候使用。
